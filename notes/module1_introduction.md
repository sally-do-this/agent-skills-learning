## 1.1 什么是Agent Skills?

### 🎯 核心定义

**Agent Skills** 是包含指令的**文件夹**，用于扩展AI Agent的能力。

```
想象一下：
❌ 传统方式：每次都要告诉Agent如何分析Excel文件
✅ 使用Skills：Agent自动加载"Excel分析技能"，知道该做什么
```

### 📦 技能的本质

```python
Skill = {
    "形式": "文件夹",
    "内容": "Markdown指令",
    "作用": "知识传递",
    "特点": "可重用、按需加载"
}
```

### 🔄 工作流程

```
用户请求
    ↓
Agent识别需求
    ↓
加载相关Skill
    ↓
执行任务
    ↓
返回结果
```

## 1.2 为什么Skills很重要?

### 💡 解决的核心问题

**问题1: 重复提示词**
```markdown
❌ 每次对话都要说：
"请按照以下步骤分析数据：
1. 检查数据完整性
2. 统计摘要
3. 可视化趋势
..."

✅ 使用Skills：
Agent自动加载"数据分析标准流程"技能
```

**问题2: 上下文浪费**
```markdown
❌ 每次都发送完整的2000行指令文档
✅ Skills只在需要时加载相关部分（渐进式披露）
```

**问题3: 知识不可重用**
```markdown
❌ 每个项目都要重新定义工作流程
✅ 一次创建技能，到处使用（开放标准）
```

### 📊 对比分析

| 方面 | 传统提示词 | Skills |
|------|-----------|--------|
| **重用性** | 每次重写 | 一次创建，多次使用 |
| **一致性** | 难以保持 | 标准化流程 |
| **上下文** | 浪费token | 按需加载 |
| **维护** | 分散在各处 | 集中管理 |
| **协作** | 难以共享 | 易于分发 |

## 1.3 技能 vs 传统方法

### 示例：代码审查场景

#### ❌ 传统方式

```python
# 每次都要写这样的提示词
prompt = """
请审查以下Python代码：
1. 检查代码风格（PEP 8）
2. 查找常见bug
3. 评估性能
4. 检查安全问题
5. 提供改进建议

代码：
{code}
"""
```

**缺点**：
- 每次都要重复
- 难以保持一致性
- 修改需要更新所有地方

#### ✅ 使用Skills

```python
# 创建一个"Python代码审查"技能
# skill/python-reviewer/skill/SKILL.md

# 使用时只需说：
"使用python-reviewer技能审查这段代码"
```

**优点**：
- 一次定义，到处使用
- 易于维护和更新
- 团队协作友好

## 1.8 关键术语

### 📚 词汇表

| 术语 | 定义 |
|------|------|
| **Skill** | 包含指令的文件夹，扩展Agent能力 |
| **Agent** | AI代理，能够执行任务的智能体 |
| **Progressive Disclosure** | 渐进式披露，按需加载信息 |
| **SKILL.md** | 技能定义文件，包含指令 |
| **MCP** | Model Context Protocol，模型上下文协议 |
| **Subagent** | 子代理，具有独立上下文的Agent |
| **Tool** | 工具，用于获取数据或执行操作的函数 |
