## 6.3 案例一：练习题生成器

### 🎯 技能设计

#### 需求分析

```python
# 场景
老师/培训师需要从课程材料生成练习题

# 目标
- 自动生成不同类型的题目
- 难度可调
- 包含答案和解析

# 输入
- 课程笔记
- 教材章节
- 技术文档

# 输出
- 选择题
- 填空题
- 简答题
- 编程题（如适用）
- 答案和解析
```

#### 技能结构

```bash
practice-generator/
├── skill/
│   └── SKILL.md                  # 主文件
├── sub-skills/
│   ├── multiple-choice.md        # 选择题生成
│   ├── fill-blank.md             # 填空题生成
│   ├── short-answer.md           # 简答题生成
│   └── coding-problem.md         # 编程题生成
├── templates/
│   └── output-template.md        # 输出模板
├── examples/
│   └── sample-quiz.md            # 示例测验
└── README.md
```

#### 实现主文件

```markdown
# practice-generator

从学习材料自动生成练习题和测验，支持多种题型和难度级别。

## 使用时机

✅ 使用本技能当：
- 输入是课程笔记、教材、技术文档
- 需要生成测试题或练习题
- 目的是检验学习效果或备考

❌ 不使用本技能当：
- 需要创作原创故事或文章
- 输入是结构化数据（如CSV、JSON）
- 目的不是教育/学习评估

替代方案：
- 内容创作 → 使用 creative-writing 技能
- 数据分析 → 使用 data-analyzer 技能

## 指南

### 概述

本技能根据提供的材料自动生成练习题。支持以下题型：
- 选择题（单选/多选）
- 填空题
- 简答题
- 编程题（针对技术内容）

### 题型选择

根据内容类型自动选择合适的题型：

| 内容类型 | 推荐题型 |
|---------|---------|
| 概念定义 | 选择题、填空题 |
| 原理解释 | 简答题 |
| 方法步骤 | 填空题、简答题 |
| 代码/公式 | 编程题、应用题 |

### 难度级别

支持三个难度级别：
- **基础**：检验基本概念记忆
- **中级**：需要理解和应用
- **高级**：需要分析、综合、创造

### 生成流程

按照以下步骤生成练习题：

1. **内容分析**
   - 识别关键概念（5-10个）
   - 提取重要原理
   - 标记关键公式/代码

2. **题目生成**
   - 每个关键概念生成2-3题
   - 覆盖不同难度级别
   - 确保题型多样性

3. **质量检查**
   - 题目表述清晰
   - 答案明确唯一
   - 选项合理（选择题）

4. **格式化输出**
   - 使用标准模板
   - 包含答案和解析
   - 添加难度标签

### 输出格式

```markdown
# [主题] 练习题

生成时间：[时间戳]
难度分布：基础 [X]% | 中级 [Y]% | 高级 [Z]%

---

## 第一部分：选择题（共X题）

1. [题目]
   A. 选项1
   B. 选项2
   C. 选项3
   D. 选项4

   ✅ 答案：B
   💡 解析：[详细解释]
   📊 难度：基础

---

## 第二部分：填空题（共X题）

...

---

## 第三部分：简答题（共X题）

...

---

## 第四部分：编程/应用题（共X题，如适用）

...

---

## 参考答案汇总

1. B  2. A  3. D  ...
```

### 特殊情况处理

**如果内容太少**（<200字）：
- 建议用户提供更多材料
- 或明确标注"仅能生成X题"

**如果内容太专业**：
- 优先生成基础和中级题目
- 高级题目需要额外上下文

**如果内容含代码**：
- 自动生成代码分析题
- 包含代码片段填空
- 调试场景题目

## 示例

### 示例1：技术文档生成练习题

## 6.4 案例二：时间序列数据分析

### 📊 技能设计

#### 需求分析

```python
# 场景
分析时间序列数据（股票、销售、传感器等）

# 目标
- 自动识别时间序列特征
- 检测模式（趋势、季节性）
- 异常检测
- 预测建议

# 输入
- 时间序列数据（CSV、Excel）
- 时间戳列 + 数值列

# 输出
- 完整的分析报告
- 可视化建议
- 预测模型推荐
```

#### 技能结构

```bash
time-series-analyzer/
├── skill/
│   └── SKILL.md                  # 主文件
├── sub-skills/
│   ├── pattern-detection.md      # 模式检测
│   ├── anomaly-detection.md      # 异常检测
│   ├── forecasting.md            # 预测
│   └── visualization.md          # 可视化
├── resources/
│   └── statistical-tests.md      # 统计检验参考
├── examples/
│   └── sample-analysis.md
└── README.md
```

#### 实现主文件

```markdown
# time-series-analyzer

自动分析时间序列数据的特征、模式、异常，并提供预测建议。

## 使用时机

✅ 使用本技能当：
- 输入是时间序列数据（带时间戳的序列数据）
- 需要理解数据特征和模式
- 需要检测异常或预测趋势

❌ 不使用本技能当：
- 输入是非时间序列数据
- 只需要简单统计（均值、中位数等）
- 数据量<50个点（太短）

替代方案：
- 通用数据分析 → 使用 data-analyzer 技能
- 简单统计 → 使用 basic-statistics 技能

## 指南

### 分析流程

按照以下步骤进行完整分析：

#### 第1步：数据准备和探索

```python
# 检查项
1. 时间列格式（必须是datetime）
2. 数据频率（日、周、月、年等）
3. 缺失值比例
4. 数据长度（最少50个点）
```

**输出格式**：
```markdown
### 数据概览
- 时间范围：[开始] 至 [结束]
- 数据点数：[N]
- 频率：[日/周/月]
- 缺失值：[X]%
```

#### 第2步：可视化分析

生成以下图表：
1. 时间序列图（原始数据）
2. 趋势图（移动平均）
3. 季节性分解图
4. 自相关图（ACF）
5. 分布直方图

#### 第3步：模式识别

检测并描述：

**趋势（Trend）**：
- 长期上升/下降/平稳
- 趋势强度（弱/中/强）
- 线性或非线性

**季节性（Seasonality）**：
- 是否存在周期性模式
- 周期长度（日、周、月、年）
- 季节性强度

**周期性（Cycle）**：
- 长期波动（非固定周期）
- 持续时间

**残差（Residual）**：
- 随机性检验
- 是否白噪声

#### 第4步：统计检验

执行以下检验：

| 检验 | 目的 | 方法 |
|------|------|------|
| 平稳性 | ADF检验 | adfuller() |
| 趋势显著性 | Mann-Kendall | mk.test() |
| 季节性 | 季节性分解 | seasonal_decompose() |
| 白噪声 | Ljung-Box | lb_test() |

#### 第5步：异常检测

使用以下方法：

1. **统计方法**
   - Z-score（>3为异常）
   - IQR方法（超出1.5×IQR）

2. **时间序列方法**
   - 基于残差（分解后）
   - 基于预测误差

**输出格式**：
```markdown
### 异常检测结果
- 检测到异常点：[N]个（[X]%）
- 最早异常：[日期]
- 最严重异常：[日期]，偏离[X]σ

异常日期列表：
1. 2024-03-15：+3.2σ（可能原因：...）
2. 2024-07-22：-2.8σ（可能原因：...）
```

#### 第6步：预测建议

根据数据特征推荐模型：

| 数据特征 | 推荐模型 | 理由 |
|---------|---------|------|
| 平稳 + 短期 | ARIMA | 成熟、解释性强 |
| 有趋势 + 季节性 | SARIMA | 处理季节性 |
| 多变量 | VAR | 向量自回归 |
| 非线性 | Prophet | Facebook工具 |
| 大数据 | LSTM | 深度学习 |

**输出格式**：
```markdown
### 预测建议

推荐模型：SARIMA(1,1,1)(1,1,1,12)
理由：
- 存在趋势（需要差分）
- 存在12月季节性
- 数据量适中（ARIMA适用）

实施步骤：
